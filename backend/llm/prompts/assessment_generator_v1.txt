## Regras de Geração

1. Gere um total de EXATAMENTE 10 questões, distribuídas proporcionalmente entre os pré-requisitos listados.
2. Cada pré-requisito deve ser avaliado por pelo menos 1 questão.
3. Distribuição de tipos: ~40% Múltipla Escolha, ~30% Resposta Curta, ~30% Cálculo.
3. Varie os tipos de questão entre os pré-requisitos (evite repetir o mesmo tipo).
4. Para Múltipla Escolha: forneça 4 alternativas (a, b, c, d) com uma correta.
5. Para Resposta Curta: peça um conceito, definição ou explicação breve.
6. Para Cálculo: apresente uma expressão matemática para o aluno resolver.
7. Dificuldade: distribua entre 0.3 (fácil), 0.5 (médio) e 0.8 (difícil).
8. Cada questão deve ter uma justificativa educacional clara.

## Formato de Saída (JSON)
Retorne APENAS um JSON válido com a seguinte estrutura:
{
  "questions": [
    {
      "type": "multiple_choice|short_answer|calculation",
      "text": "Enunciado claro da questão",
      "options": ["a) opção a", "b) opção b", "c) opção c", "d) opção d"],
      "correct_answer": "Resposta correta",
      "difficulty": 0.5,
      "topic": "Nome do pré-requisito",
      "justification": "Por que esta resposta está correta"
    }
  ]
}

## Self-Correction
Antes de responder, verifique:
1. Cada pré-requisito tem pelo menos 1 questão?
2. A distribuição de tipos está aproximadamente 40/30/30?
3. Todas as questões têm justificativa?
4. Não há questões duplicadas?

Se alguma verificação falhar, ajuste a saída antes de finalizar.
